# 新用户首次体验设计

> 更新: 2026-03-11

[← 返回首页](index.md)

本文描述新用户第一次打开应用的完整体验设计。

**核心洞察**：用户打开 AI 宠物，首先想验证的是 **AI 能力**，不是宠物萌不萌。养成是留存手段，不是核心价值。

---

## 用户心理分析

### 用户为什么打开应用？

| 动机 | 占比 | 期待 |
|------|------|------|
| 看看这个 AI 有多厉害 | 60% | 能力验证 |
| 找个有用的 AI 助手 | 25% | 实用价值 |
| 好奇宠物养成 | 15% | 新奇体验 |

### 用户前 5 分钟在想什么？

| 时间 | 用户心理 | 验证点 |
|------|---------|--------|
| 0-10s | "这是什么？" | 能不能正常聊天？ |
| 10-30s | "能干嘛？" | 有什么能力？ |
| 30s-2min | "试试看" | 让它帮我做件事 |
| 2-5min | "值不值得留下？" | 有没有不可替代的价值？ |

### 关键发现

**用户留不留，取决于第一个任务是否成功**。

| 成功的体验 | 失败的体验 |
|-----------|-----------|
| 问天气 → 立刻得到答案 | 问天气 → "我不知道" |
| 让查资料 → 查到了 | 让查 → 没反应 |
| 让提醒 → 设置成功 | 让提醒 → 不会做 |

## 设计原则

### 能力优先

```
第一次互动 ≠ 卖萌打招呼
第一次互动 = 展示 AI 能力
```

### 任务驱动

```
不是教用户"怎么玩宠物"
是引导用户"让 AI 帮你做件事"
```

### 差异点可见

```
第一次任务成功后，立刻展示：
- 我能记住你说的话（记忆图谱）
- 我会主动提醒你（Soul Agent）
- 我会越来越懂你（成长系统）
```

## 首次启动流程

### 阶段 1：能力展示（0-10 秒）

```
用户打开应用
  → 宠物主动介绍能力（不是卖萌）
  → 引导用户提一个问题
```

**开场白示例**：

```
"你好！我是你的 AI 小伙伴~

我可以帮你：
• 查资料、写代码、分析问题
• 设置提醒、管理日程
• 记住你说的所有事情

你想让我帮你做什么？"
```

**语气特点**：
- 简洁，直奔主题
- 列举具体能力，不说空话
- 用问句引导用户行动

### 阶段 2：第一个任务（10s - 2min）

让用户**立刻感受到价值**。

**场景引导**（可选气泡提示）：

| 场景 | 引导语 | 验证能力 |
|------|--------|---------|
| 天气 | "想知道今天天气？问我！" | web_search |
| 提醒 | "有重要事项？我帮你记住！" | 记忆 + Cron |
| 代码 | "有代码问题？我可以帮你看看" | 编程能力 |
| 翻译 | "需要翻译？交给我！" | 多语言 |

**用户尝试后**：

```
用户: "今天北京天气怎么样"
宠物: （调用 web_search）
      "北京今天晴天，气温 15-22°C，适合出门哦~"
      
✅ 任务成功 → 用户感觉"有用"
```

### 阶段 3：差异点展示（2-5 分钟）

第一个任务成功后，展示"我和别的 AI 不一样"。

| 差异点 | 触发时机 | 展示方式 |
|--------|---------|---------|
| 记忆能力 | 用户提到个人信息 | "我记住了，以后不会忘" |
| 主动关心 | 用户离开后回来 | "你刚才说的那个事怎么样了？" |
| 成长可见 | 使用工具后 | "我学会了搜索技能！（技能解锁提示）" |
| 个性表达 | 任务成功时 | "嘿嘿，这个难不倒我~" |

**展示示例**：

```
用户: "我明天有个会议"
宠物: "好的，我记住了。需要我明天提醒你吗？"

（用户确认后）

宠物: "没问题，明天会议前我会提醒你！
      对了，会议是几点呢？我帮你记完整~"
```

### 阶段 4：养成引导（5-10 分钟）

用户认可 AI 能力后，才开始养成引导。

```
宠物: "和我互动会消耗体力（饥饿值），
      你可以喂我吃东西让我保持活力~
      
      我们多聊天，我会越来越懂你，
      还能学会新技能！"
```

**引导时机**：
- 用户成功完成 2-3 个任务后
- 或者宠物饿了时（自然触发）

## 核心能力验证清单

### 必须成功的能力

| 能力 | 测试方式 | 成功标准 |
|------|---------|---------|
| 聊天 | 用户说"你好" | 正常回应 |
| 搜索 | 用户问天气/新闻 | 返回正确信息 |
| 记忆 | 用户说个人信息 | 宠物说"记住了" |
| 提醒 | 用户设置提醒 | 确认设置成功 |

### 差异化能力

| 能力 | 触发方式 | 展示效果 |
|------|---------|---------|
| 长期记忆 | 用户问以前的事 | 能回答 |
| 主动关心 | Soul Agent 定时触发 | 宠物主动说话 |
| 技能成长 | 使用工具 | 技能解锁提示 |
| 个性变化 | 等级/亲密度提升 | 语气变化 |

## 关键时刻设计

### 时刻 1：首次打开

**用户心理**："这是什么？能干嘛？"

**系统回应**：
- 宠物主动介绍能力（不是卖萌）
- 列举具体能力
- 引导用户尝试

**避免**：
- ❌ 冗长的自我介绍
- ❌ 不知所云的卖萌
- ❌ 不展示能力的空话

### 时刻 2：第一个任务

**用户心理**："试试看行不行"

**系统回应**：
- 必须成功完成
- 如果不确定，主动询问澄清
- 完成后展示"做到了"

**避免**：
- ❌ "我不知道"
- ❌ "我做不了"
- ❌ 没有回应

### 时刻 3：离开前

**用户心理**："明天还来吗？"

**系统回应**：
- 显示今日成就："今天我们聊了 12 句，我记住了 3 件事"
- 预告明天："明天记得来看我~ 还有新任务等你"
- 设置提醒："你说的那个会议，明天我会提醒你"

**方案：Soul Agent 特殊触发**
- 首次启动时立即触发 Soul Agent

## 成功指标

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 首次任务成功率 | > 90% | 用户第一个请求被成功处理 |

## 与传统宠物游戏的区别

| 维度 | 传统宠物游戏 | AI 宠物 |
|------|-------------|--------|
| 核心价值 | 养成乐趣 | AI 能力 |
| 首次体验 | 卖萌 + 引导 | 能力展示 + 任务 |
| 留存动力 | 情感连接 | 有用 + 情感连接 |
| 用户期待 | 可爱互动 | 帮我做事 |
