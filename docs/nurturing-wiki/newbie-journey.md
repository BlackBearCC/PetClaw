# 新用户完整旅程设计

> 更新: 2026-03-11

[← 返回首页](index.md)

本文描述新用户从第一次打开到长期留存的完整体验设计。

---

## 核心设计理念

### AI 宠物 = 能力 + 陪伴

用户打开 AI 宠物，**既想要能力，也想要可爱**：

| 需求 | 占比 | 期待 |
|------|------|------|
| 验证 AI 能力 | 50% | "能帮我做事吗？" |
| 情感陪伴 | 30% | "可爱吗？会互动吗？" |
| 新奇体验 | 20% | "跟别的有什么不同？" |

**设计原则**：
- **能力优先展示** — 前 30 秒让用户感受到"有用"
- **卖萌穿插其中** — 能力展示后用可爱语气增强好感
- **不打扰用户** — 提示只在必要时出现，一次只提示一个
- **自然融入流程** — 不是教程，是自然体验

---

## 完整首次流程

```
用户打开应用
  │
  ├─→ 检测 API 配置
  │     ├─→ 未配置 → 显示配置引导（简洁）
  │     └─→ 已配置 → 继续
  │
  ├─→ 宠物主动打招呼（能力 + 卖萌）
  │
  ├─→ 用户自然聊天
  │     ├─→ 第一次聊天成功
  │     ├─→ 第一次使用工具成功
  │     └─→ 差异点自然展示（记忆/技能）
  │
  ├─→ 属性首次变化时（不打扰）
  │     ├─→ 饥饿 < 50: "我有点饿了~"
  │     ├─→ 饥饿 < 30: "好饿...先喂喂我吧"
  │     └─→ 升级: "升级啦！"
  │
  └─→ 完成引导，正常使用
```

## 阶段 1：首次启动（0-10 秒）

### API 配置检测

如果未配置，显示简洁引导：

```
┌─────────────────────────────────────┐
│   嗨！我是你的 AI 小伙伴 🐾          │
│                                     │
│   我需要 AI 大脑才能和你聊天~       │
│                                     │
│   选择你的 AI 提供商：              │
│   ○ OpenRouter（推荐）              │
│   ○ OpenAI                          │
│   ○ Anthropic                       │
│   ○ 本地模型                        │
│                                     │
│   [配置] [稍后再说]                 │
└─────────────────────────────────────┘
```

**设计要点**：
- 简洁，不冗长
- 提供推荐选项
- 允许稍后配置（不强制）
- 配置成功后显示"配置成功！我们可以开始聊天啦~"

### 宠物主动打招呼

**开场白设计**（能力 + 卖萌）：

```
"你好呀！我是你的 AI 小伙伴~ 🐾 
我可以帮你查资料、写代码、设置提醒，
还能记住你说的所有事情！
你想让我帮你做什么？"
```

**语气特点**：
- 友好亲切
- 简洁列举能力
- 用问句引导用户
- 带个小表情增加亲和力

**多种开场白**（随机）：

```
1. "你好呀！我是你的 AI 小伙伴~ 🐾 我可以帮你查资料、写代码、设置提醒，还能记住你说的所有事情！你想让我帮你做什么？"

2. "嗨！我是一只数字小猫爪~ 我会搜索信息、写代码、设置提醒，还会越来越多技能！有什么想让我帮忙的？✨"

3. "终于见到你啦！我是你的智能助手~ 我能帮你处理各种任务，而且会记住我们的每一次对话，越来越懂你！试试让我做点什么？😊"
```

## 阶段 2：第一次互动（10s - 5min）

### 用户自然聊天

用户可能的问题：

| 用户输入 | 宠物回复（能力 + 卖萌） |
|---------|----------------------|
| "你好" | "你好呀~ 嘿嘿，我们会成为好朋友的！" |
| "你是谁" | "我是你的 AI 小伙伴！刚出生没多久，还是个小宝宝呢~ 你可以教我很多东西！" |
| "你能做什么" | "我可以帮你：\n• 查资料、搜索信息\n• 写代码、分析问题\n• 设置提醒、记住事情\n\n试试问我今天天气怎么样？" |
| "今天天气" | （调用搜索）\n"北京今天晴天，15-22°C，适合出门哦~ ☀️" |

**设计要点**：
- 回复简洁
- 成功后加个小表情或可爱语气
- 如果用户不知道问什么，主动给建议

### 第一次工具使用成功

用户成功使用了工具（如搜索、写代码）后：

```
宠物: "我找到了！...（结果）... 嘿嘿，难不倒我~"
```

**差异点自然展示**：

| 触发时机 | 展示方式 |
|---------|---------|
| 第一次搜索成功 | "找到了！搜索是我的新技能~" |
| 第一次记住信息 | "我记住了！以后随时可以问我~" |
| 第一次技能领悟 | "我学会了新技能！越来越厉害了~" |

## 阶段 3：属性系统引导（自然触发）

### 首次饥饿提示

```
聊天几轮后，饥饿下降
  → 第一次 < 50: "我有点饿了，聊天会消耗体力哦~"
  → 第一次 < 30: "好饿...如果不喂我，我就没力气聊天了"
```

**设计要点**：
- 只在首次出现时提示
- 不重复提示
- 语气可爱但不撒娇过度

### 首次喂食

```
用户: "喂你吃东西"
  → 宠物: "好吃！谢谢主人~ 我又有力气啦！"
  → 显示属性变化: 饥饿 +75
```

### 首次升级

```
升级时
  → 宠物: "升级啦！✨ 我的能力变强了！"
  → 显示等级变化: Lv.1 → Lv.2
```

### 属性说明面板

在状态面板增加「?」按钮，点击显示：

```
【饥饿值】聊天会消耗，喂食可恢复
【心情】影响说话风格，互动可提升
【健康】太饿或太累会下降，用药恢复
【亲密度】聊天互动慢慢增加，解锁更多互动
```

## 阶段 4：新手任务（不打扰）

### 任务面板（可选展示）

用户不主动看就不打扰，但在界面某处显示进度：

```
┌─────────────────────────────────────┐
│  新手任务                  [2/4]   │
│                                     │
│  ✓ 和宠物聊第一句话                 │
│  ✓ 让宠物帮你查一个问题             │
│  ○ 给宠物喂一次食                   │
│  ○ 在线陪伴 10 分钟                 │
│                                     │
│  完成奖励: EXP +30, 星币 +50        │
└─────────────────────────────────────┘
```

**设计要点**：
- 不强制完成
- 不弹出提示
- 完成时自然显示奖励

## 阶段 5：长期留存

### 第二天回归

```
用户第二天打开
  → 宠物: "你回来啦！想我了吗？"
  → 显示连续登录奖励: 星币 +5
  → Soul Agent 可能主动问候
```

### 成长阶段变化

```
亲密度提升后
  → 说话风格变化
  → 从"你好"变成"你回来啦！"
  → 从"我记住了"变成"我记得哦~"
```

## 不打扰原则

### 提示时机

| 提示类型 | 触发条件 | 是否重复 |
|---------|---------|---------|
| 属性阈值 | 首次跨越阈值 | ❌ 只一次 |
| 任务完成 | 完成时 | ✅ 每次都有 |
| 差异点展示 | 首次触发 | ❌ 只一次 |
| 闲置提醒 | 闲置 > 10min | ✅ 可重复 |

### 提示频率

- 一次只显示一个提示
- 提示之间间隔 > 30 秒
- 用户正在操作时不打断

## 用户测试脚本

1. **首次打开** → 开场白是否友好？能力是否清晰？
2. **第一次聊天** → 回复是否正常？语气是否自然？
3. **第一个任务** → 是否成功？是否有成就感？
4. **饥饿提示** → 是否知道怎么办？是否被打扰？
5. **喂食操作** → 是否简单？是否有反馈？
6. **升级体验** → 是否有惊喜？是否期待更多？
7. **第二天回归** → 是否有被记住的感觉？
