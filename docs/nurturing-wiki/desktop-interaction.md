# 桌面交互系统

[← 返回首页](index.md)

## 设计原则

宠物不是"浮在桌面的窗口"，而是**住在你桌面上的生物**。它感知窗口、任务栏、鼠标、屏幕边缘，并与之发生物理关系。

---

## 窗口感知

### 窗口边缘站立

宠物能"看见"其他窗口的边界，站在窗口顶部边缘。

```
┌─────────────────────┐
│  VS Code             │
│                      │  😺 ← 站在窗口顶上
├──────────────────────┤
│  code code code      │
│  code code code      │
```

- 通过 Win32 API (`EnumWindows` / `GetWindowRect`) 获取可见窗口列表
- 宠物可站/走/坐在任意窗口的顶部边缘
- 窗口移动/关闭时宠物跟随或跳下
- 窗口最大化时宠物跳到任务栏或桌面底部

### 任务栏栖息

宠物可以坐在任务栏上，像坐在架子上一样。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  🪟  📁  🌐  💬       😺        🔊 🕐
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 检测任务栏位置和高度（支持上/下/左/右）
- 在任务栏区域上方行走/坐下
- 用户点击任务栏图标时宠物可能被"吓一跳"

### 屏幕边缘交互

| 行为 | 触发 | 动画 |
|------|------|------|
| 探头 | 宠物走到屏幕边缘 | 身体藏在边缘后面，只露出头 |
| 爬上来 | 从屏幕下方/侧边 | 双手扒住边缘，慢慢爬上来 |
| 挂壁 | 走到屏幕侧边 | 像壁虎一样贴在边上 |
| 滑落 | 挂壁一段时间后 | 慢慢往下滑，抓不住 |

## 鼠标交互

### 追鼠标

当鼠标静止超过 3 秒且在宠物附近（200px 内），宠物好奇地走向鼠标。

```
状态机:
  idle → 检测鼠标距离 < 200px 且静止 > 3s
       → curious_walk (朝鼠标方向走)
       → 到达鼠标位置 → sniff (闻一闻) → idle
       → 鼠标移动 → startled (吓一跳) → idle
```

- 不能太频繁（CD 30s），否则烦人
- 鼠标突然移走时播放惊吓动画
- 用户忙碌打字时不触发（检测键盘活跃度）

### 躲鼠标

低心情时，鼠标靠近宠物，宠物会主动躲开。

### 玩鼠标

高心情时，宠物会试图"拍"鼠标指针（像猫拍激光笔）。

## 窗口互动

### 推窗口

宠物走到某个窗口边缘，做出推的动作，窗口微微移动几像素。

```
触发条件:
  - 宠物处于 playful 状态
  - 随机触发 (概率 5%，CD 10min)
  - 目标窗口非全屏、非置顶

效果:
  - 窗口水平移动 3-8px
  - 宠物播放 push 动画
  - 推完坐下休息
```

- 轻微到不影响工作，但能被注意到
- 可在设置中关闭

### 窥屏

宠物走到窗口边缘，转身看向窗口内容，做出好奇/无聊的表情。

### 拍窗口

宠物站在窗口顶部，用爪子拍窗口标题栏。纯动画，无实际效果。

## 物理系统 (简化版)

### 重力

宠物默认受重力影响，会落到最近的"平台"上：

```
平台优先级:
  1. 当前站立的窗口顶部
  2. 下方最近的窗口顶部
  3. 任务栏顶部
  4. 屏幕底部 (地面)
```

### 跳跃

宠物可以在不同高度的窗口之间跳跃。

```
跳跃条件:
  - 目标窗口顶部与当前平台高度差 < 200px
  - 水平距离 < 300px

动画: 蓄力 → 抛物线跳跃 → 着陆
```

### 掉落

窗口关闭/移开时，宠物脚下平台消失 → 掉落动画 → 落到下一个平台。

## 拖拽物理

当前拖拽只是移动位置。增加物理反馈：

| 拖拽行为 | 反应 |
|---------|------|
| 抓起 | 宠物四肢张开，表情惊讶 |
| 快速移动 | 宠物表情变晕 |
| 甩动 | 宠物眼睛变螺旋 |
| 放下 | 轻放 → 站稳；高处扔下 → 掉落+着陆 |
| 扔向屏幕边缘 | 宠物从边缘滑出去，过几秒从另一边爬回来 |

### Electron 实现要点

```
- 透明无框窗口 (现有)
- 窗口大小需要覆盖宠物可活动范围 (不能只是宠物大小)
  → 方案 A: 全屏透明窗口，宠物在内部移动 (简单但覆盖点击)
  → 方案 B: 窗口跟随宠物移动 + 扩大碰撞区域 (现有方案改进)
- 点击穿透: 非宠物区域设置 pointer-events: none
- 鼠标位置: screen.getCursorScreenPoint() 轮询 (100ms)
- 窗口列表: 通过 Node addon 或 child_process 调用系统 API
```

### 性能约束

- 平台扫描: 每 2s 一次，不能更频繁
- 鼠标追踪: 100ms 轮询，仅在需要时开启
- 宠物物理: 和渲染同帧 (requestAnimationFrame)
- 总 CPU 占用目标: < 2%
