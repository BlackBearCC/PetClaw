## 一、数值养成提示系统

### 问题

新用户不理解：
- 饥饿值低了会怎样？
- 心情有什么用？
- 为什么要喂食？
- 属性之间有什么关系？

### 设计方案

#### 1. 首次触发提示

| 触发点 | 提示内容 |
|--------|---------|
| 首次饥饿 < 50 | "我有点饿了，聊天会消耗体力哦。记得喂我吃东西~" |
| 首次饥饿 < 30 | "好饿...如果不喂我，我就没力气陪你聊天了" |
| 首次心情 < 50 | "我有点不开心...陪我玩玩？" |
| 首次健康 < 50 | "我有点不舒服...能给我吃点药吗？" |
| 首次属性回满 | "状态恢复了！感觉活力满满~" |
| 首次升级 | "升级啦！我的能力变强了，还能解锁新功能！" |

#### 2. 属性说明面板

在客户端状态面板增加「?」按钮，点击显示：

```
【饥饿值】
• 聊天会消耗饥饿值
• 饥饿值低于 30 无法聊天
• 喂食可以恢复

【心情】
• 影响我的说话风格
• 和我互动、玩耍可以提升
• 心情好时，我会更活泼

【健康】
• 太饿或太累会下降
• 使用药物恢复
• 健康不好时，其他属性恢复变慢

【亲密度】
• 聊天、互动会慢慢增加
• 亲密度高时，我会更懂你
• 解锁更多互动方式
```

#### 3. 核心循环可视化

在首次体验中增加一个简单的循环图：

```
┌─────────────────────────────────────┐
│          核心玩法循环               │
│                                     │
│    聊天 ──→ 消耗饥饿 ──→ 喂食       │
│      │                    │        │
│      ↓                    ↓        │
│   获得经验 ←─────── 完成任务       │
│      │                              │
│      ↓                              │
│    升级 ──→ 解锁新能力             │
│                                     │
└─────────────────────────────────────┘
```

## 二、API 配置引导

### 问题

新用户打开应用后：
1. 不知道需要配置 API Key
2. 不知道去哪里配置
3. 不知道配置什么模型
4. 配置失败不知道怎么办

### 设计方案

#### 1. 首次启动检测

```typescript
// 检测 API 是否已配置
async function checkApiConfigured(): Promise<boolean> {
  // 检查是否有可用的 LLM provider
  const providers = await getAvailableProviders();
  return providers.length > 0;
}
```

#### 2. 配置引导流程

```
首次启动
  → 检测 API 配置
  → 未配置 → 显示配置引导
  → 已配置 → 正常启动
```

**方案 B：网页引导**

#### 4. 模型推荐

| 用户类型 | 推荐方案 | 说明 |
|---------|---------|------|
| 小白用户 | OpenRouter auto | 自动选择，简单 |
| 注重成本 | OpenRouter + 便宜模型 | 如 gemini-2.0-flash |
| 注重质量 | OpenRouter + claude-3.5-sonnet | 效果好 |
| 隐私优先 | 本地 Ollama | 数据不出本机 |

#### 5. 错误处理

| 错误 | 提示 |
|------|------|
| API Key 无效 | "API Key 无效，请检查是否复制完整" |
| 余额不足 | "账户余额不足，请先充值" |
| 网络错误 | "网络连接失败，请检查网络或使用代理" |
| 模型不可用 | "该模型暂时不可用，试试其他模型？" |

#### 6. 配置状态 RPC

```typescript
// 获取配置状态
"character.config.status": () => ({
  configured: boolean,
  provider?: string,
  model?: string,
  needsSetup: boolean,
})

// 获取配置向导
"character.config.wizard": () => ({
  providers: [
    { id: "openai", name: "OpenAI", url: "https://platform.openai.com" },
    { id: "anthropic", name: "Anthropic", url: "https://console.anthropic.com" },
    { id: "openrouter", name: "OpenRouter", url: "https://openrouter.ai", recommended: true },
    { id: "ollama", name: "本地模型", url: "http://localhost:11434" },
  ],
})

// 测试配置
"character.config.test": async ({ provider, apiKey, model }) => ({
  success: boolean,
  error?: string,
})
```

## 三、完整首次流程（更新）

```
用户打开应用
  │
  ├─→ 检测 API 配置
  │     ├─→ 未配置 → 显示配置引导
  │     │              ├─→ 配置成功 → 继续
  │     │              └─→ 配置失败 → 显示帮助
  │     └─→ 已配置 → 继续
  │
  ├─→ 宠物主动打招呼（展示能力）
  │
  ├─→ 显示新手任务面板
  │     ├─→ 和宠物聊第一句话
  │     ├─→ 让宠物帮查一个问题
  │     ├─→ 喂宠物吃东西
  │     └─→ 在线陪伴 10 分钟
  │
  ├─→ 属性首次变化时显示提示
  │     ├─→ 饥饿 < 50: "有点饿了"
  │     ├─→ 饥饿 < 30: "好饿，要喂我"
  │     └─→ 升级: "升级啦！"
  │
  └─→ 完成引导，正常使用
```
